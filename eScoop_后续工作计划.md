# eScoop / 乐生活 后续工作计划

日期：2026-07-06

## 一、项目宪法确认

项目的长期方向是：

> 做一个 AI 驱动的北美华人本地生活平台，同时成为本地商家的 AI 营销获客工具。

核心原则：

1. Merchant-first：先服务商家获客。
2. AI-first：AI 是商家发布和营销效率工具。
3. Local-first：聚焦洛杉矶等北美华人本地生活场景。
4. Mobile-first：移动端优先。

项目不是单纯复制小红书，也不是优先做纯社区。社区内容是后续增强，第一阶段应围绕商家增长和内容供给来做。

1. 用户访问页面时，只读取 Supabase `deal_current_prices`。

3. AI 只生成中文摘要和推荐理由，不生成价格。
4. Walmart、Target、Macy's 优先通过具体商品页公开结构化价格抓取。

6. Costco、Sam's Club 使用实验抓取，自动写入缓存但保留人工修正入口。
7. Best Buy 有 API Key 时走官方 API，否则可用公开商品页。
8. 普通 `fetch` 失败后，GitHub Actions 使用 Playwright Chromium 读取 JS 渲染后的价格。
9. Walmart、Target、Costco 等商家的抓取规则进入 `RETAILER_ADAPTERS`，后续参考 GitHub 项目时只扩展配置。
10. 搜索页抓到的价格、实验商家价格和弱匹配价格会写入 `review_flags`，用于后续人工复核。
11. 4.53 修复了 `playwright_config`、`target_scraper_config`、`costco_product_tracker` 等来源误回退 `manual_cache` 的问题。
12. 4.54 参考 `aransaseelan/CostcoPriceTracker` 的 Product ID 思路，为 Costco 增加候选 URL 重试。
13. 4.54 确认 `walmart-sales-forecasting` 属于销量预测/机器学习方向，不作为实时价格数据源。
14. 4.54 起 GitHub 版本目录推荐使用英文路径 `versions/4.54`，减少 Actions 路径问题。
15. 4.55 将外部来源拆为三类：固定商品页、官方周广告、官方限时优惠。
16. 4.55 Best Buy 使用官方 Deal of the Day，Walmart 使用官方 Flash Deals。
17. 4.55 ALDI 使用官方 Weekly Ads，Target 使用官方 Weekly Ad，并按每周刷新处理。
18. 4.55 牛奶、大米、Costco 牛排等刚需商品使用具体商品页每日刷新。
19. 页面始终保留“官网核对”，价格和库存以商家官网最终显示为准。

## 三、近期原则

近期开发不做大重构。

必须遵守：

1. 保持现有 UI 风格，不突然换设计。
2. 新功能优先接在现有页面和现有函数体系里。
3. 不引入复杂推荐算法、私信增强、完整 CRM、自动投流。
4. 所有页面尽量三步内完成核心任务。
5. 商家进入后台后的第一目标是“发布内容”。

## 三点五、版本规则

从本轮开始按以下规则管理版本：

1. 当前新增商家 AI 发布内容版本命名为 3.1。
2. 之后如果只是调整视觉或修复问题，修改第三位，例如 3.11、3.12。
3. 如果继续增加新内容，修改第二位，例如 3.20、3.30。
4. 如果做大版本更新或架构迁移，命名为 4.0。

## 四、第一阶段 MVP 目标

第一阶段要跑通这条链路：

> 商家注册 / 认证 → 商家主页 → 上传图片 + 一句话 → AI 生成多平台文案 → 复制 / 保存 / 发布 → eScoop 信息流展示 → 简单数据统计。

需要输出的 AI 文案版本：

1. 小红书文案
2. Instagram 文案
3. Facebook 文案
4. Google Business 文案
5. eScoop 站内版本

语气规则：

- 真实
- 简洁
- 有帮助
- 不夸大
- 不制造虚假承诺

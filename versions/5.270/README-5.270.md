# 乐生活 v5.270 - 商家员工岗位与订单分工

## 订单岗位

- **店主 / 店经理**：完整订单、后厨、传菜、收银、菜单、餐桌和团队权限。
- **后厨**：只进入后厨订单屏；将菜品从红色“完成制作”标成绿色“已完成”，也可取消完成。
- **传菜**：只进入传菜上桌屏；只看后厨已完成的菜品，可上桌或撤回上桌。
- **收银**：可查看订单、核销优惠券和会员奖励、完成现金或刷卡订单并生成账单。
- **运营、清洁、其他**：可按需保留内容运营或岗位身份；清洁和其他为后续排班预留。

## 矩阵账号

- 店主在团队管理中可搜索乐生活用户后邀请。
- 一个员工可同时配置多个岗位。
- 岗位自动带上必要权限，店主还能单独增加“查看账单、管理菜单、管理餐桌”等细项权限。
- 数据库会强制校验权限，不能仅靠网页按钮绕过。

## 验收建议

1. 用店主账号邀请测试用户，分别设为后厨、传菜、收银。
2. 后厨账号只确认制作完成；传菜账号仅看到完成制作的菜并上桌。
3. 收银账号可以结单和核销，后厨账号不能。
4. 给同一测试账号勾选“后厨 + 传菜”，确认可使用两张工作屏。

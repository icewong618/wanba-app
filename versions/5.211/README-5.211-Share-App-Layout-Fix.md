# 5.211 分享链接与 App 笔记体验修复

## 已完成

- 分享笔记链接直接打开时，先展示加载状态并单独读取目标笔记，避免首页数据尚未到达时出现可穿透的白屏。
- 首页绿色快捷栏在手机 App 中改为五项均分，确保“视频”入口不会被右侧裁切。
- 笔记详情页的顶部、底部操作栏加入横向安全区和最小宽度限制，避免分享、评论等操作在窄屏中被挤出画面。
- 图片轮播限制在详情页容器内，不再因 WebView 的视口宽度计算造成右侧裁切。
- 竖屏 YouTube 容器改为固定 9:16 并在容器内居中，去掉此前使用 `100vw` 平移带来的上方、左侧黑边和偏移。

## 建议测试

1. 将任意笔记分享到微信、Facebook 或浏览器，再通过分享链接直接打开，确认先出现加载状态后进入该笔记，而不是白屏。
2. 在 iPhone 与 Android App 首页确认绿色快捷栏的“视频”完整可见且可以点击。
3. 打开含多张图片的笔记，确认图片不会裁切、可左右滑动，顶部分享按钮与底部回复/互动按钮均可点击。
4. 打开竖屏 YouTube 笔记，确认视频居中、上下没有异常黑边，评论区仍可上下滚动。

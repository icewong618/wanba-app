# v5.167 原生 App 稳定性与按需权限版

## 修复

- Android 封装 App 首页的首行笔记向下调整 14px，不再被固定头部遮挡。
- 首页远程数据到达前改为骨架卡片，不再短暂显示“正在读取内容”。
- App 壳会保留 `post` 与 `user` 分享参数，避免分享内容进入 App 后丢失目标页面。
- 分享 Worker 从 GitHub Raw 回源改为 jsDelivr 的固定提交 CDN，避免 GitHub Raw `429` 导致公开笔记、个人主页或商家主页白屏。
- iOS 封装 App 中的 Facebook 分享改走系统分享面板；系统将把链接交给用户选择的 Facebook、微信等应用。

## 已接入的原生能力

- 系统分享：`@capacitor/share`
- 触感反馈：`@capacitor/haptics`
- 本地通知：`@capacitor/local-notifications`
- 既有能力：相机、相册、定位、通讯录。

个人页左上角菜单新增“App 权限与提醒”，可在封装 App 内逐项测试。

## 权限规则

- 仅浏览不申请权限。
- 发布时才申请相机或相册。
- 点击附近商家、附近会员卡时才申请使用期间定位。
- 点击邀请通讯录好友时才申请通讯录。
- 用户打开本地提醒时才申请通知权限。
- 触感反馈和系统分享不需要运行时权限。

## 原生工程更新

- Android 已增加通知和定时本地提醒声明。
- iOS 保留使用期间定位，移除不必要的始终定位说明。
- 已运行 `npx cap sync` 同步 iOS 与 Android 原生工程；需要重新在 Xcode / Android Studio 构建并安装测试包。

## 尚未在本版启用

- 远程推送：需要 FCM / APNs 服务器端配置。
- 桌面角标：需要额外原生插件，且部分 Android 桌面不支持。
- Face ID / 指纹登录：需要安全存储并在下一版单独接入，不能把登录信息保存在网页缓存中。

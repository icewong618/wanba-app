# v5.510 - Production Engineering Closure

本版本完成公测前的工程收口，并修复独立省钱模块的请求地址错误。

## 修复

- 公共模块运行时同时支持相对 API 路径与完整 Supabase URL，避免重复拼接域名导致省钱页面无法加载。
- 更新省钱、本周、消息三个独立页面的运行时版本，确保 App WebView 不会继续读取旧缓存。

## 工程收口

- 样式拆分为基础样式、移动端安全区样式、商家样式。
- 构建流程会生成完整 `_site/` 发布目录，并压缩 HTML/CSS 静态资源。
- 发布时复制全部独立模块目录，避免部署后缺少模块文件。
- 清理主控制器中被后续同名函数覆盖的 35 处旧实现。
- 发布检查会阻止重复全局函数、缺失样式分层、入口页不同步和本地模块缺失。

## 兼容性

- 保持现有网页链接、App WebView 路由和 Supabase 数据字段不变。
- `index.html` 与 `app.html` 保持一致。

## 验证

- `npm run check:release`
- 主控制器与构建脚本语法检查
- 省钱模块完整 Supabase URL 请求检查
